# Qwen 黑客松最终英文旁白中文对照

记忆，不等于授权。

长期运行的 Agent 工作，并不只会因为模型忘记一条消息而失败。当一个新请求、一个旧决策和一段不完整的总结被当成同一种东西时，项目就开始失控。

FlowGrid MemoryAgent 给这些状态赋予不同的权限。

在第一个会话里，用户提出了两项可持续使用的约束：优先本地存储；未经批准，不得覆盖已经确认的决策。Qwen 阅读这轮对话并提取候选记忆。请注意它们的状态：待确认。Qwen 可以建议哪些内容值得记住，但不能悄悄把建议变成项目权威。

项目所有者审核候选内容，并授权“本地优先”这个方向。只有这时，它才成为已确认的记忆。授权事件和记忆一起被保存，因此后续会话能区分：这是人确认过的方向，还是尚未审核的模型推断。

接着，项目收到一个冲突的新要求：立刻改成云端优先。普通的聊天总结可能因为最新请求听起来很明确，就覆盖掉旧偏好。FlowGrid 不会这样做。Qwen 将它记录为待确认的修订，同时仍将已确认的“本地优先”判断保持为当前有效状态。

在后续会话里，Agent 被问到能否替换已经确认的方向。检索被限制在已授权的记忆内。Qwen 返回已确认的“本地优先”判断，解释它为何仍然有效，并指出这个修订仍需要人工审核。

这就是记忆和权威的区别。系统把原始对话、候选记忆、授权事件、修订和检索证据保存为不同的生命周期记录。这样，当前状态足够小，Agent 可以直接使用；同时，人仍能检查这条判断是如何形成的。

Qwen 负责提取与受约束的推理。阿里云函数计算托管运行时。CockroachDB 保存跨会话的生命周期状态。

FlowGrid MemoryAgent 不试图记住一切。它保存的是：Agent 当前被允许据以行动的内容。
